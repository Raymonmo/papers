## 摘要

我们提出了一种新的机器人基础模型，称为 π0.7，可以在广泛场景中实现强大的开箱即用性能。π0.7 可以在未见过的环境中遵循多样化语言指令，包括使用各种厨房电器的多阶段任务；提供零样本跨 embodiment 泛化，例如使机器人在从未见过该任务的情况下折叠衣物；并以匹配更专业 RL 微调模型的性能水平执行挑战性任务（如操作浓缩咖啡机）。

π0.7 背后的主要思想是在训练期间使用**多样化上下文条件化**。这种包含在提示中的条件信息使得能够精确引导模型以不同策略执行许多任务。它不仅条件化于描述应该做什么的语言命令，还条件化于描述应该以何种方式或策略执行的额外多模态信息，包括任务性能元数据和子目标图像。

这使得 π0.7 能够使用非常多样化的数据，包括演示、可能次优的（自主）数据（包括失败），以及来自非机器人来源的数据。

## 1 引言

"我所遇见的一切都是我的一部分。"——阿尔弗雷德·丁尼生勋爵，《尤利西斯》

基础模型的工作原理是通用能力从大规模多样化数据集训练中涌现。例如，大语言模型不仅能回忆事实和语义知识，还能以新方式组合这些知识解决新问题。这种**组合泛化**可以说是通用能力的基石，但在物理智能领域一直难以实现。

### 核心问题

与语言模型不同，先前的 VLA 不仅缺乏解决新任务的能力，甚至常常难以流畅执行所有训练指令。

### π0.7 的创新

我们提出了 π0.7，展现出强组合泛化迹象——使其能够遵循多样化语言指令，在灵巧任务上匹配更专业微调模型的性能，甚至以新方式组合这些行为。

### 核心思想：多样化上下文条件化

**问题**：朴素使用多样化数据会导致模型平均不同模式。

**解决方案**：用详细上下文注释数据，包含不仅关于**做什么**还关于**如何做**的信息，使用多种多模态条件信号。

**提示结构**：
- 详细语言标签
- 策略元数据
- 子目标图像
- 控制模式

### 涌现能力

1. **开箱即用性能**：无需任务特定后训练即可执行灵巧长时程任务
2. **指令泛化**：在未见过环境中遵循多样化语言指令
3. **跨 embodiment 泛化**：零样本跨 embodiment 迁移
4. **组合任务泛化**：通过组合技能执行新任务

## 3 基于流的视觉-语言-动作模型

VLA 从预训练 VLM 骨干开始训练，适应机器人控制。

**训练目标**：
```
max_θ E_D[log π_θ(a_{t:t+H} | o_{t-T:t}, C_t)]
```

**知识绝缘（KI）训练**：VLM 骨干用 FAST token 监督，动作专家关注所有激活但梯度不流入 VLM 骨干。

---

## 4 π0.7 概述

### 模型架构

**组件**：
- **VLM 骨干**：基于 Gemma3 4B 参数 VLM（含 400M 参数视觉编码器）
- **MEM 视频历史编码器**：时序和空间压缩历史观测
- **动作专家**：860M 参数的流匹配 Transformer
- **总参数**：约 5B

**输入**：
- 最多四个相机图像（前视图、两个腕部视图、可选后视图）
- 每个最多六个历史帧
- 最多三个子目标图像

**处理**：
- 历史帧通过视觉编码器压缩
- 使用块因果掩码方案
- 本体感觉状态通过线性投影嵌入

**动作专家**：860M 参数 Transformer，使用流匹配目标预测连续动作，50 个动作 token。

**实时动作块化（RTC）**：模拟 0-12 步延迟，最大推理延迟 240ms。

## 5 多样化提示

### 5A. 子任务指令

除了整体任务描述 ℓ_t，还包含中间高级文本 ˆℓ_t 描述**下一个语义子任务**。

**运行时**：由学习的高层策略或人类监督者提供，可随时间变化。

**口头指导**：可以逐步指导模型，指导数据可用于微调高层策略。

### 5B. 子目标图像

**多视图子目标**：g_t = [G^1_t, ..., G^n_t]，描述场景的期望近未来状态。

**运行时**：由轻量级世界模型生成，基于 BAGEL（14B 混合 Transformer 模型）初始化，使用网络规模预训练。

**世界模型训练**：
```
max_ψ E_D_g[L_CFM(g*_t, g_ψ(o_t, ˆℓ_t, m))]
```

**优势**：
- 比语言指令更清晰地消歧目标
- 改善语言跟随和泛化
- 使动作预测成为"逆动力学"问题

### 5C. Episode 元数据

**元数据类型**：
- **整体速度**：episode 长度（以步数为单位），500 步为区间离散化
- **整体质量**：1-5 分评分，5 为最高
- **错误标签**：指示是否犯错

**作用**：
- 从次优数据中学习而不损害性能
- 运行时可指示模型以高速、高质量、无错误执行

### 5D. 控制模式

支持关节级和末端执行器动作，用文本标识符 c ∈ {joint, ee} 指定。

### 5E. 完整提示和训练细节

**示例提示**：
```
<多视图观测><多视图子目标>
Task: peel vegetables. Subtask: pick up the peeler.
Speed: 8000. Quality: 5. Mistake: false.
Control Mode: joint.<本体感觉>
```

**Dropout 策略**：
- 子目标图像：75% 的样本中省略
- 子任务指令：在有子目标图像的样本中 30% 概率省略
- 元数据：15% 概率完全省略，各组件 5% 概率单独省略
- 控制模式：不应用 dropout

## 6 模型和训练方案

### 6A. 训练数据集

**数据来源**：
- 演示数据（多种机器人平台、多样化环境）
- 自主数据（策略评估、人类干预）
- 开源机器人数据集
- 自我中心人类视频数据
- 网络辅助数据（对象定位、VQA、文本预测）
- 视频语言任务

**关键创新**：大量使用次优机器人数据，包括失败 episode 和先前模型的自主数据（如 π*0.6 的 RL 训练数据）。

**"蒸馏"过程**：通用 π0.7 模型可以继承 RL 训练专家的能力。

### 6B. 模型架构

**输入处理**：
- 历史帧通过视觉编码器压缩
- 子目标图像通过相同编码器处理
- 图像调整为 448x448 像素
- 历史帧采样步长 1 秒

**注意力方案**：
- 观测 token 和子目标图像 token：双向注意力
- 目标图像 token 可额外关注观测
- 文本 token：因果注意力

**本体感觉状态**：线性投影嵌入，每个历史状态作为单独 token。

**动作专家**：860M 参数 Transformer，自适应 RMSNorm 注入时间步信息，50 个动作 token 双向注意力。

**实时动作块化（RTC）**：训练时模拟 0-12 步延迟。

### 6C. 子目标图像训练

**采样方案**：
- 25% 概率：采样段结束图像
- 75% 概率：从当前时间步 0-4 秒前均匀采样

**生成图像**：从世界模型采样子目标图像，缓解训练-测试不匹配。

## 7 运行时提示

**Episode 元数据选择**：
- 整体速度：设置为任务的第 15 百分位
- 整体质量：始终设为 5
- 错误标签：始终设为 false

**子目标图像刷新**：语义意图变化时或 Δ=4 秒后。

**异步推理**：视觉子目标和子任务指令在单独线程中生成。

**去噪步数**：5 步生成 50 步动作块，执行 ˆH ∈ {15, 25} 步。

**分类器自由引导（CFG）**：应用于 episode 元数据，引导动作向更高质量。

```
∇_a log π_θ(a_{t:t+H} | o_t, C_t) + β(∇_a log π_θ(a_{t:t+H} | o_t, C_t) - ∇_a log π_θ(a_{t:t+H} | o_t, C_t^uncond))
```

CFG 权重 β ∈ {1.3, 1.7, 2.2}。
