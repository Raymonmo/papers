## 🎯 一句话总结

**EXPO-FT 提出了一种样本高效的 RL 微调系统，通过冻结的基策略 + 轻量编辑策略 + Q 值选择机制，在仅约 19 分钟的真实世界数据内将预训练 VLA 模型微调至近完美性能。**

---

## 🔍 研究背景与动机

### 为什么需要 EXPO-FT？

1. **预训练 VLA 的局限**：VLA 模型（如 OpenVLA、π0）在多样化任务上展示了强大泛化能力，但在实际部署中**可靠性不足**
2. **现有 RL 方法的不足**：
   - 从零训练的 RL：不利用预训练先验，样本效率低
   - 直接微调 VLA：难以达到实际所需的样本效率和成功率
3. **核心挑战**：如何在保持预训练知识的同时，通过 RL 高效适应新任务

---

## 💡 核心贡献

1. **EXPO-FT 系统**：稳定、样本高效的 VLA 策略 RL 微调框架
2. **编辑策略机制**：冻结基 VLA，用轻量网络预测动作修正，避免大模型梯度回传
3. **OTF（On-the-Fly）动作选择**：从基动作和编辑动作中选 Q 值最高的
4. **人类在环干预**：训练初期人类纠正，随策略改进逐步减少
5. **开源代码库**

## 📖 方法详解

### 3.1 整体架构

```
┌─────────────────────────────────────────────────┐
│                  EXPO-FT 系统                    │
├─────────────────────────────────────────────────┤
│                                                  │
│  预训练 VLA（冻结）  →  基动作 a（8 个 chunk）    │
│         +                                        │
│  编辑策略 π_edit    →  修正量 â（8 个 chunk）     │
│         ↓                                        │
│  编辑动作 ã = a + â（8 个 chunk）                │
│         ↓                                        │
│  Q 网络评估 → 从 16 个候选中选最优 ã*             │
│         ↓                                        │
│  执行 ã*                                         │
└─────────────────────────────────────────────────┘
```

### 3.2 双策略设计

**基策略 π_VLA**：
- 预训练的流匹配 VLA 模型
- **训练期间完全冻结**，保留预训练知识
- 每次决策采样 8 个随机动作 chunk

**编辑策略 π_edit**：
- 轻量级网络，预测对基动作的小幅修正
- 修正量约束在 [-β, β] 范围内
- 目标：在基动作附近探索更优动作

```
最终动作 = 基动作 a + 编辑修正 â
```

### 3.3 OTF 动作选择

```
候选集 = {a_1, ..., a_8, ã_1, ..., ã_8}  共 16 个

ã* = argmax Q_φ(s, a)  over all 16 candidates

选择 Q 值最高的动作执行（确定性选择，非 softmax）
```

**关键优势**：
- 不需要对大 VLA 做梯度回传
- Q 网络可以评估基动作和编辑动作的质量
- 动态选择最优动作，兼顾预训练知识和探索

### 3.4 Q 网络架构（Critic）

**RedQ 风格集成**：
- 10 个 Q 网络的集成
- 每次随机选 2 个，取最小值作为目标
- 使用层归一化

**视觉编码**：
- 独立的轻量 ResNet-50（不共享 VLA 编码器）
- 阶段深度 (3,4,6,3)，64 个滤波器
- 输出 512 维图像嵌入

**本体感觉**：
- 嵌入到 64 维
- 与视觉特征融合

**MLP**：
- 3 个隐藏层，宽度 256

### 3.5 时序扩展动作（Action Chunks）

- 编辑策略和 Q 函数都操作在**动作 chunk**（长度 H）上
- 每次重新规划执行 C ≤ H 步
- 保持现代 VLA 的时序抽象设计

### 3.6 人类在环干预

```
训练初期：人类频繁干预（通过 SpaceMouse）
    ↓
策略改进：干预频率逐渐降低
    ↓
训练后期：干预降至零
```

- 人类可以覆盖动作 chunk 中的任意时间步
- 纠正数据存入回放缓冲区
- 提供有针对性的监督信号，减少探索需求

### 3.7 训练流程

```
步骤1：评估零样本 VLA 性能
       如果不足 → 收集演示 + 监督微调至 ~40% 成功率

步骤2：用离线数据初始化回放缓冲区

步骤3：在线 RL 训练
       Learner-Actor 解耦架构
       Learner：处理 VLA 训练/推理
       Actor：处理环境交互
       支持同步/异步通信
```

### 3.8 奖励设计

- **稀疏二元奖励**（0 或 1）
- 基于规则的分类器，准确率 >95%
- 任务特定检测器：
  - 糖果检测：像素颜色匹配
  - 方块拾取：高度阈值

## 📊 实验结果

### 实验任务

| 任务 | 描述 | 难度 |
|------|------|------|
| String Light Routing | 将灯串穿过挂钩 | 高精度、柔性物体 |
| Pool Ball Pocketing | 将台球打入袋 | 动态操作 |
| Flower Insertion | 将花插入花瓶 | 高精度装配 |
| Plug Insertion | 将插头插入插座 | 精密装配 |

### 核心结果

| 指标 | 结果 |
|------|------|
| **成功率** | 30/30（100%）所有任务 |
| **平均训练时间** | 19.1 分钟在线数据 |
| **对比基线** | 优于 RL-from-scratch 和 VLA 微调方法 |

### 消融实验

| 方法 | 成功率 |
|------|--------|
| RL from scratch | 低 |
| VLA 直接微调 | 中 |
| **EXPO-FT** | **100%** |

## ⚠️ 局限性与未来方向

### 局限
1. **依赖预训练 VLA 质量**：基策略太差时需要先监督微调
2. **稀疏奖励假设**：需要任务特定的奖励分类器
3. **任务范围**：主要验证在桌面操作任务

### 未来方向
1. 扩展到更复杂的长时任务
2. 探索自动化的奖励设计
3. 与更多 VLA 架构的兼容性

---

## 💬 个人评价与启发

### 优点
1. **巧妙的架构设计**：冻结 VLA + 轻量编辑策略，避免大模型梯度回传
2. **极高样本效率**：19 分钟达到 100% 成功率
3. **实用性强**：人类在环 + 稀疏奖励，降低部署门槛
4. **理论清晰**：OTF 选择机制优雅地融合预训练知识和 RL 探索

### 启发
1. **预训练 + RL 微调是 VLA 部署的关键路径**
2. **编辑策略模式**可能成为 VLA RL 微调的标准范式
3. **人类在环的价值**：少量人类纠正可以极大加速 RL 收敛
