## 摘要

我们提出了一族新的策略梯度强化学习方法，该方法在通过与环境交互采样数据和使用随机梯度上升优化"代理"目标函数之间交替进行。标准策略梯度方法对每个数据样本执行一次梯度更新，而我们提出了一种新颖的目标函数，使得可以进行多个 epoch 的小批量更新。我们将这种新方法称为近端策略优化（PPO），它具有信赖域策略优化（TRPO）的一些优势，但实现起来更简单、更通用，并且具有更好的样本复杂度（根据经验）。我们的实验在一系列基准任务上测试了 PPO，包括模拟机器人运动和 Atari 游戏，结果表明 PPO 优于其他在线策略梯度方法，并且总体上在样本复杂度、简洁性和实际运行时间之间取得了良好的平衡。

## 1 引言

近年来，针对使用神经网络函数逼近器的强化学习，已经提出了几种不同的方法。主要的竞争者包括深度 Q 学习 [Mni+15]、"朴素"策略梯度方法 [Mni+16] 以及信赖域/自然策略梯度方法 [Sch+15b]。然而，在开发一种可扩展（到大型模型和并行实现）、数据高效且稳健（即在各种问题上无需超参数调优即可成功）的方法方面，仍有改进空间。

Q 学习（带函数逼近）在许多简单问题上失败¹，且其理论理解不足；朴素策略梯度方法的数据效率和稳健性较差；而信赖域策略优化（TRPO）相对复杂，且与包含噪声（如 dropout）或参数共享（在策略和价值函数之间，或与辅助任务之间）的架构不兼容。

本文试图通过引入一种算法来改善当前状况，该算法在仅使用一阶优化的情况下，达到了 TRPO 的数据效率和可靠性能。我们提出了一种带有裁剪概率比率的新目标函数，该函数对策略性能形成了悲观估计（即下界）。为了优化策略，我们在从策略采样数据和对采样数据执行多个 epoch 优化之间交替进行。

我们的实验比较了代理目标函数各种不同版本的性能，发现带有裁剪概率比率的版本表现最好。我们还将 PPO 与文献中的几种先前算法进行了比较。在连续控制任务上，它比我们比较的算法表现更好。在 Atari 上，它的表现显著优于 A2C（在样本复杂度方面），与 ACER 相似，尽管它更简单得多。

¹ 虽然 DQN 在像 Arcade Learning Environment [Bel+15] 这样的离散动作空间游戏环境中效果很好，但它尚未在连续控制基准（如 OpenAI Gym [Bro+16] 中的那些，以及 Duan 等人 [Dua+16] 描述的那些）上展示出良好性能。

## 2 背景：策略优化

### 2.1 策略梯度方法

策略梯度方法通过计算策略梯度的估计量并将其代入随机梯度上升算法来工作。最常用的梯度估计量形式为：

`ĝ = Ě_t[∇_θ log π_θ(a_t|s_t) · Â_t]` 　　　(1)

其中 π_θ 是随机策略，Â_t 是时间步 t 的优势函数估计量。这里的期望 Ě_t[...] 表示在有限样本批次上的经验平均值，该算法在采样和优化之间交替进行。使用自动微分软件的实现通过构造一个目标函数来工作，该目标函数的梯度就是策略梯度估计量；估计量 ĝ 通过对以下目标求导获得：

`L^PG(θ) = Ě_t[log π_θ(a_t|s_t) · Â_t]` 　　　(2)

虽然对同一个轨迹上的这个损失 L^PG 执行多步优化看起来很吸引人，但这样做没有充分的理论依据，而且根据经验，它常常导致破坏性的大策略更新（见第 6.1 节；结果未展示，但与"无裁剪或惩罚"设置相似或更差）。

### 2.2 信赖域方法

在 TRPO [Sch+15b] 中，在策略更新大小的约束下最大化一个目标函数（"代理"目标）。具体来说：

**最大化**_θ

`Ě_t[(π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)) · Â_t]` 　　　(3)

**约束条件**：

`Ě_t[KL[π_{θ_old}(·|s_t), π_θ(·|s_t)]] ≤ δ` 　　　(4)

这里 θ_old 是更新前的策略参数向量。在对目标进行线性近似并对约束进行二次近似后，可以使用共轭梯度算法有效地近似求解此问题。

证明 TRPO 合理性的理论实际上建议使用惩罚而非约束，即求解无约束优化问题：

**最大化**_θ

`Ě_t[(π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)) · Â_t - β · KL[π_{θ_old}(·|s_t), π_θ(·|s_t)]]` 　　　(5)

对于某个系数 β。这源于以下事实：某个代理目标（在状态上计算最大 KL 而非平均值）构成了策略 π 性能的下界（即悲观界）。TRPO 使用硬约束而非惩罚，因为很难选择一个在不同问题上——甚至在单个问题中，其特征在学习过程中会发生变化——都表现良好的单一 β 值。

因此，为了实现我们的一阶算法模拟 TRPO 单调改进的目标，实验表明简单地选择固定惩罚系数 β 并用 SGD 优化惩罚目标公式 (5) 是不够的；需要额外的修改。

## 3 裁剪代理目标

令 r_t(θ) 表示概率比率：

`r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)`

因此 r(θ_old) = 1。TRPO 最大化"代理"目标：

`L^CPI(θ) = Ě_t[(π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)) · Â_t] = Ě_t[r_t(θ) · Â_t]` 　　　(6)

上标 CPI 指的是保守策略迭代 [KL02]，该目标就是在那里提出的。如果没有约束，最大化 L^CPI 将导致过大的策略更新；因此，我们现在考虑如何修改目标，以惩罚使 r_t(θ) 远离 1 的策略变化。

我们提出的主要目标如下：

`L^CLIP(θ) = Ě_t[min(r_t(θ)·Â_t, clip(r_t(θ), 1-ε, 1+ε)·Â_t)]` 　　　(7)

其中 epsilon 是一个超参数，例如 ε = 0.2。

这个目标的动机如下。min 内的第一项是 L^CPI。第二项 clip(r_t(θ), 1-ε, 1+ε)Â_t 通过裁剪概率比率来修改代理目标，这移除了将 r_t 移出区间 [1-ε, 1+ε] 的激励。最后，我们取裁剪和未裁剪目标的最小值，因此最终目标是未裁剪目标的下界（即悲观界）。通过这种方案，我们只在概率比率的变化会使目标改善时忽略它，而在它使目标变差时包含它。

注意 L^CLIP(θ) 在 θ_old 附近（即 r = 1 处）与 L^CPI(θ) 一阶相同，然而，当 θ 远离 θ_old 时它们变得不同。

图 1 绘制了 L^CLIP 中的单一项（即单个时间步）；注意概率比率 r 在 1-ε 或 1+ε 处被裁剪，取决于优势是正还是负。

**图 1**：显示代理函数 L^CLIP 作为概率比率 r 的函数的单一项（即单个时间步）的图，左图为正优势，右图为负优势。每个图上的红点显示优化的起始点，即 r = 1。注意 L^CLIP 对许多这样的项求和。

图 2 提供了关于代理目标 L^CLIP 的另一种直觉来源。它展示了当我们沿策略更新方向（通过对连续控制问题进行近端策略优化——即我们稍后将介绍的算法——获得的）插值时，几个目标如何变化。我们可以看到 L^CLIP 是 L^CPI 的下界，对过大的策略更新进行了惩罚。

**图 2**：代理目标，我们在初始策略参数 θ_old 和更新后的策略参数之间插值，更新后的参数是我们在一次 PPO 迭代后计算得到的。更新后的策略与初始策略的 KL 散度约为 0.02，这是 L^CLIP 最大化的点。该图对应于 Hopper-v1 问题上的第一次策略更新，使用第 6.1 节提供的超参数。

## 4 自适应 KL 惩罚系数

另一种方法——可以用作裁剪代理目标的替代方案，或与之结合使用——是对 KL 散度使用惩罚，并调整惩罚系数，使我们在每次策略更新时达到 KL 散度的某个目标值 d_targ。

在我们的实验中，我们发现 KL 惩罚比裁剪代理目标表现更差，然而，我们在这里包含了它，因为它是一个重要的基线。在该算法的最简单实例中，我们在每次策略更新中执行以下步骤：

- 使用多个 epoch 的小批量 SGD，优化 KL 惩罚目标：

`L^{KLPEN}(θ) = Ě_t[(π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)) · Â_t - β · KL[π_{θ_old}(·|s_t), π_θ(·|s_t)]]` 　　　(8)

- 计算 d = Ě_t[KL[π_{θ_old}(·|s_t), π_θ(·|s_t)]]
  - 如果 d < d_targ / 1.5，则 β ← β/2
  - 如果 d > d_targ × 1.5，则 β ← β × 2

更新后的 β 用于下一次策略更新。使用这种方案，我们偶尔会看到 KL 散度与 d_targ 显著不同的策略更新，但这些情况很少见，β 会快速调整。上面的 1.5 和 2 是启发式选择的，但算法对它们不太敏感。β 的初始值是另一个超参数，但在实践中并不重要，因为算法会快速调整它。

## 5 算法

前面各节的代理损失可以通过对典型策略梯度实现进行微小更改来计算和求导。对于使用自动微分的实现，只需构造损失 L^CLIP 或 L^{KLPEN} 代替 L^PG，并对该目标执行多步随机梯度上升。

大多数计算方差缩减的优势函数估计量的技术使用学习到的状态价值函数 V(s)；例如，广义优势估计 [Sch+15a]，或 [Mni+16] 中的有限时域估计量。如果使用在策略和价值函数之间共享参数的神经网络架构，我们必须使用结合策略代理和价值函数误差项的损失函数。

这个目标可以进一步通过添加熵奖励来增强，以确保充分探索，如过去的工作 [Wil92; Mni+16] 所建议。结合这些项，我们得到以下目标，每次迭代（近似地）最大化：

`L^{CLIP+VF+S}_t(θ) = Ě_t[L^CLIP_t(θ) - c₁·L^VF_t(θ) + c₂·S[π_θ](s_t)]` 　　　(9)

其中 c_1, c_2 是系数，S 表示熵奖励，L^{VF}_t 是平方误差损失 (V_θ(s_t) - V_t^{targ})²。

一种策略梯度实现风格——在 [Mni+16] 中推广，非常适合与循环神经网络一起使用——运行策略 T 个时间步（其中 T 远小于 episode 长度），并使用收集的样本进行更新。这种风格需要一个不超出时间步 T 的优势估计量。[Mni+16] 使用的估计量是：

`Â_t = -V(s_t) + r_t + γ·r_{t+1} + ... + γ^{T-t+1}·r_{T-1} + γ^{T-t}·V(s_T)` 　　　(10)

其中 t 指定给定长度 T 轨迹段中的时间索引，范围在 [0, T] 内。

推广这个选择，我们可以使用广义优势估计的截断版本，当 λ = 1 时简化为公式 (10)：

`Â_t = δ_t + (γλ)·δ_{t+1} + ... + (γλ)^{T-t+1}·δ_{T-1}` 　　　(11)

其中

`δ_t = r_t + γ·V(s_{t+1}) - V(s_t)` 　　　(12)

下面展示了一个使用固定长度轨迹段的近端策略优化（PPO）算法。每次迭代，每个 N 个（并行）执行器收集 T 个时间步的数据。然后我们在这些 NT 个时间步的数据上构造代理损失，并用小批量 SGD（或通常为了更好的性能，使用 Adam [KB14]）对其进行 K 个 epoch 的优化。

---

**算法 1：PPO，Actor-Critic 风格**

```
for iteration = 1, 2, ... do
    for actor = 1, 2, ..., N do
        在环境中运行策略 π_{θ_old}，持续 T 个时间步
        计算优势估计 Â_1, ..., Â_T
    end for
    使用 K 个 epoch 和小批量大小 M ≤ NT 优化代理 L 关于 θ
    θ_old ← θ
end for
```

### 6.1 代理目标的比较

首先，我们在不同超参数下比较几种不同的代理目标。这里，我们将代理目标 L^CLIP 与几种自然变体和消融版本进行比较。

**无裁剪或惩罚**：L_t(θ) = r_t(θ) Â_t

**裁剪**：L_t(θ) = min(r_t(θ)Â_t, clip(r_t(θ), 1-ε, 1+ε)Â_t)

**KL 惩罚（固定或自适应）**：L_t(θ) = r_t(θ)Â_t - β·KL[π_{θ_old}, π_θ]

对于 KL 惩罚，可以使用固定惩罚系数 β，也可以使用第 4 节描述的自适应系数（使用目标 KL 值 d_targ）。注意我们也尝试了在对数空间中裁剪，但发现性能没有更好。

因为我们为每种算法变体搜索超参数，所以我们选择了一个计算成本低的基准来测试算法。具体来说，我们使用了 OpenAI Gym [Bro+16] 中实现的 7 个模拟机器人任务²，这些任务使用 MuJoCo [TET12] 物理引擎。我们在每个任务上训练一百万个时间步。

## 7 结论

我们引入了近端策略优化，这是一族策略优化方法，使用多个 epoch 的随机梯度上升来执行每次策略更新。这些方法具有信赖域方法的稳定性和可靠性，但实现起来简单得多，只需对朴素策略梯度实现进行几行代码更改，适用于更一般的设置（例如，当使用策略和价值函数的联合架构时），并且具有更好的整体性能。
