# ConRFT 读后报告

**论文**：ConRFT: A Reinforced Fine-tuning Method for VLA Models via Consistency Policy
**作者**：Yuhui Chen, Shuai Tian 等（中科院自动化所）
**发表**：arXiv 2025

---

## 一、论文定位

ConRFT 的核心思路：借鉴 LLM/VLM 领域中 RL 微调的成功经验，把 RL 引入 VLA 模型微调。核心创新是**统一训练目标**和**一致性策略动作头**，其他组件（Cal-QL、HIL 机制）都是已有技术的组装。

## 二、核心贡献

### 1. 统一训练目标

离线和在线阶段用同一个 loss 结构：

```
L_π = β·L_BC + η·L_Q
```

- L_BC：行为克隆损失，锚定策略在演示动作附近
- L_Q：Q 函数引导，推向高价值动作
- 区别仅在于 β 和 η 的权重不同（在线阶段 β 减小，η 增大）

好处：离线阶段学到的 Q 函数在线阶段直接复用，策略平滑过渡。

### 2. 一致性策略动作头

替代扩散策略和 MLP：
- 比扩散策略更轻量（推理只需一步）
- 比 MLP 表达力更强（能处理不一致的演示数据）
- 作为 VLA 模型和 RL 微调之间的桥梁

### 3. 两阶段设计

**阶段一：Cal-ConRFT（离线微调）**
- 基于 Cal-QL（校准 Q 学习）
- 用 20-30 个演示训练
- BC 损失 + Q 损失，对 OOD 动作做保守惩罚
- 输出：初步策略 + 初步 Q 函数

**阶段二：HIL-ConRFT（在线微调）**
- 策略与真实环境交互
- 人类通过 SpaceMouse 纠正，纠正数据存入 D
- 对称采样：D（50%）+ R（50%）
- β 减小，η 增大
- 输出：最终策略（96.3% 成功率）

## 三、实验结果

| 方法 | 成功率 | episode 长度 |
|------|--------|-------------|
| SFT（纯监督） | 39.4% | 57.5 步 |
| HG-DAgger | 65% | 56.3 步 |
| PA-RL | 71.3% | 51.1 步 |
| **HIL-ConRFT** | **96.3%** | **30.7 步** |

关键发现：
- 微调 VLA（96.3%）远优于从头训练 RL（31.9%）
- 20 个演示 + 在线 RL 优于 150 个演示的纯监督学习
- 跨 VLA 模型通用：在 Kosmos-2 和 PaliGemma 上均达 100%

## 四、关键问答

### Q：两个阶段微调的都是 VLA 模型吗？

是，但**只微调动作头**，VLA 的视觉编码器和 Transformer 骨干冻结。原因：
- 实时性能（冻结骨干推理更快）
- 防止灾难性遗忘
- 数据效率（动作头参数少，20-30 条演示就够）
- 训练稳定性

### Q：动作头是一致性策略头吗？是在扩散策略基础上微调吗？

动作头是一致性策略头，但**不是在扩散策略基础上微调**，而是直接用一致性策略替代：
- 扩散策略：迭代去噪（10-20 步）
- 一致性策略：一步映射

两者是不同的方法，一致性策略受扩散模型启发，但跳过了去噪过程。

### Q：BC 损失的具体公式

```
L_BC = E[‖f_ψ(a + k_m·z, k_m | E(s)) - a‖²]
```

- 给演示动作 a 加不同水平的噪声 k_m·z
- 让网络学会从任意噪声水平恢复到干净动作
- 比直接模仿更鲁棒，网络学的是"去噪"而非"复制"

### Q：ConRFT 没有用 SAC？

没有。ConRFT 用的是 Cal-QL + 标准 TD 学习，没有熵正则化。BC 损失起到了类似 SAC 中熵项的作用（锚定策略，防止退化）。

### Q：离线微调 Cal-ConRFT 详细机制

基于 Cal-QL：
- BC 损失让策略模仿演示
- Q 损失让策略偏向高 Q 值动作
- Cal-QL 的保守惩罚：对 OOD 动作的 Q 值超过参考策略价值时进行惩罚

离线阶段和 SFT 成功率相同（39.4%），说明离线阶段只是初始化，真正提升来自在线阶段。

## 五、局限性

1. 奖励工程敏感性：二元分类器可能存在 reward hacking
2. 编码器和 Transformer 骨干冻结：限制了感知模块的优化
3. 任务特定性：高度特化的策略，泛化能力有限

## 六、个人启发

1. RL + VLA 的结合是可行的，通过合适的框架设计可以有效增强 VLA 模型
2. 一致性策略的优势：比扩散策略轻量，比 MLP 表达力强
3. 两阶段设计的重要性：离线初始化为在线探索提供安全起点
4. 统一训练目标的设计很优雅
