## 🎯 一句话总结

**提出 Consistency Policy，通过一致性蒸馏从预训练 Diffusion Policy 中蒸馏出单步/少步策略，推理速度提升一个数量级，同时保持竞争力的成功率，支持在笔记本 GPU 上推理。**

---

## 🔍 研究背景与动机

### 核心问题

1. **Diffusion Policy 推理慢**：需要 100 步去噪，约 1 秒/动作
2. **资源受限系统**：移动操作器、四旋翼等无法配备高端 GPU
3. **动态任务需求**：平衡物体、动态环境需要更快控制频率

### 核心思想

**一致性蒸馏**：从预训练 Diffusion Policy 蒸馏出一致性策略，沿 ODE 轨迹强制自一致性，实现单步生成。

---

## 💡 核心贡献

1. **Consistency Policy**：将一致性模型应用于机器人策略
2. **推理加速**：比最快替代方法快一个数量级
3. **笔记本 GPU 推理**：在资源受限环境中部署
4. **鲁棒性**：对预训练 Diffusion Policy 质量鲁棒

## 📖 方法详解

### 3A. 预备知识：扩散模型

**概率流 ODE（PFODE）**：
```
dx_t = [μ(x_t, t) - ½σ(t)²∇log p_t(x_t|o)] dt
```

**EDM 框架**：
```
dx_t/dt = -(x_t - s_ϕ(x_t, t; o)) / t
```

**去噪分数匹配（DSM）损失**：
```
L_DSM(θ) = E[d(x_0, s_ϕ(x_t, t; o))]
```

### 3B. 训练

**教师模型（EDM）**：
- 使用 DSM 损失训练
- 伪 Huber 损失作为度量
- Heun 二阶求解器

**学生模型（Consistency Policy）**：

**CTM 目标**：强制 ODE 轨迹上的自一致性
```
L_CTM = d(g_θ(x_s^(t), s, 0; o), g_θ(x_s^(u), s, 0; o))
```

**最终训练目标**：
```
L_CP = α·L_CTM + β·L_DSM
```

**关键设计**：
- u = t-1，s 任意低于 u
- 采样 z ~ N(0, I) 而非 N(0, T²I)
- 预热启动学生模型

### 3C. 推理

**单步推理**：
- 采样 z ~ N(0, I)
- 计算 x = g_θ(z, T, 0; o)
- 部署 x 作为动作

**3 步推理**：
- 链式生成：T → 0 → t₁ → 0 → t₂ → 0
- 优先从早中期时间步链式
- 细分离散化时间

### 3D. 实现细节

**架构**：1D 卷积 UNet（与 Diffusion Policy 相同）
- FiLM 块条件化
- 扩展 FiLM 块适应停止时间步 s
- 零初始化扩展层

## 📊 实验设置与主要结果

### 仿真任务

**Robomimic**：Lift, Can, Square, Tool Hang（图像观测）
**Push-T**：状态观测，接触丰富
**Franka Kitchen**：状态观测，长时程多阶段

### 核心结果

| 方法 | NFE | Lift | Can | Square | ToolHang | Push-T |
|------|-----|------|-----|--------|----------|--------|
| DDPM | 27 | 1.00 | .97 | .93 | .79 | .87 |
| DDiM | 9 | 1.00 | .82 | .85 | .14 | .78 |
| **CP (1步)** | **1** | 1.00 | .98 | .92 | .70 | .82 |
| **CP (3步)** | **3** | 1.00 | .95 | .96 | .77 | .84 |

**推理速度**：
- DDPM: 110ms
- DDiM: 11ms
- **CP (1步)**: **1ms**
- **CP (3步)**: **2ms**
